# Outpost Needs & Invoicing

An exercise in modelling a rather complex, multi-stepped domain, an orbital space station.  
Starting with the name itself, it doesn't abbreviate to _ONI_ by chance, and all through the domain, I pay homage to, or, another way of looking at it, shamelessly ripoff, the countless survival/builder/spaceships sandbox games I used to, and still very much, love to, play on, such at them as I might (which is a lot. I'm **really** that bad at them).

Besides being an exercise in domain modelling, the implementation is a Tour de Force of the F# language, where I try to use as many of the features that make it the excellent language it is as I can cram.

Current iteration is back-end/business logic only, no UIs, no web frameworks or REST APIs, no databases.
